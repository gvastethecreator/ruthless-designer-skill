# Distinction Pass

Use this when visual quality is in scope, the user says the UI feels generic/bland/AI-made, the task is a redesign, or the surface must compete with generic public design patterns.

Anti-slop removes obvious tells. Distinction adds the reason a reviewer would prefer this interface.

For high-ambition, "incredible", "standout", or open-ended creation, pair this file with [direction-sprint.md](direction-sprint.md) and [signature-moves.md](signature-moves.md). The sprint chooses the direction; this pass rejects reflexes and verifies the chosen move is not decoration.

## Reflex Audit

Before choosing a direction, name and reject the predictable moves:

- First reflex: the obvious AI default for this category, such as centered hero, purple gradient, three cards, icon tiles, fake product screenshot, or generic glass.
- Second reflex: the "better but still saturated" move, such as default editorial monochrome, terminal-dark developer cosplay, luxury beige, faux brutalism, or cinematic scroll effects on a task UI.
- Category costume: a visual trope that signals the industry without serving this product, audience, or workflow.
- Evidence gap: places where generic copy, fake data, decorative metadata, invented screenshots, or placeholder imagery are trying to stand in for the real artifact.

Done when the design read says what default path was rejected and what concrete alternative will carry the surface.

## Signature Moves

Pick one or two signature moves. A signature move must be visible, useful, testable, or decision-improving. It is not decoration.

- Domain artifact: make the real product object, workflow state, content type, dataset, place, material, or screenshot shape the composition.
- Task lens: organize the screen around the user's decision instead of the product's feature list.
- State craft: make empty, loading, error, permission, recovery, or first-run states feel designed, not bolted on.
- Comparative surface: show before/after, selected/unselected, risk/reward, cost/value, or current/target so the UI helps judgment.
- Interaction grammar: one repeated feedback pattern maps to the task, such as inspect, commit, compare, select, recover, or progress.
- Constraint as style: choose one hard constraint for brand surfaces, such as monochrome plus one material color, type-led editorial rhythm, image-led full bleed, dense catalog grid, or civic restraint.

Cut any move that cannot survive responsive layout, accessibility basics, and proof.

Use [signature-moves.md](signature-moves.md) when the direction needs a sharper move than these categories suggest.

## Product Vs Brand

Product distinction usually comes from trust, state coverage, information density, speed, and lower cognitive load. Surprise is expensive; spend it only where it helps the task.

Brand distinction usually comes from point of view, imagery, typography, pacing, and a memorable constraint. Restraint is valid only when it is specific enough that a competitor would not make the same page.

Hybrid surfaces need a boundary: expressive brand treatment can introduce the product, but forms, tables, settings, and repeated task controls return to product discipline.

## Replacement Moves

- Three equal feature cards -> one proof block, one workflow strip, one comparison surface, or an asymmetric grid with different information roles.
- Fake dashboard preview -> real screenshot, generated bitmap, live component state, or no preview with an explicit asset gap.
- Generic hero metadata -> a visible next section with proof, product artifact, customer outcome, or task state.
- Decorative motion -> state feedback, spatial continuity, progress, or deletion.
- Trendy palette -> palette derived from brand artifact, product domain, real image, or existing tokens.
- Cute placeholder copy -> literal action, audience, object, and consequence.
