# Golden Dashboard Direction

Use as a compact reference for product-register work. Do not copy names or layout blindly.

## Brief

Operations dashboard for a studio that schedules rooms, crews, gear, and client approvals.

## Direction

- Killed default: generic KPI cards above a table.
- Killed second reflex: dark command-center shell with decorative alerts.
- Selected direction: operator cockpit centered on today's conflicts.
- Signature move: conflict map row grammar. Every booking row exposes time pressure, missing owner, gear risk, and next action in one readable line.
- Composition: decision table plus evidence pane.
- Dials: variance 3, motion 2, density 8.

## System

- Top bar: date, saved views, search, one primary action.
- Left rail: studio rooms and view filters.
- Center: dense booking list grouped by time block.
- Right pane: selected booking evidence, approval status, gear checklist, recovery actions.
- States: empty day, loading rooms, conflict error, permission denied for finance fields, long client names, mobile compressed room tabs.

## Proof

- Desktop screenshot shows conflicts before metrics.
- Mobile screenshot keeps today's next action visible.
- Detector clean enough to avoid broad transitions, missing labels, fake data, and fixed-width overflow.
- Final claim limited if live drag/drop and slow-network states were not inspected.
