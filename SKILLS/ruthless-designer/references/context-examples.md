# Context Calibration Examples

Use these only to test whether the chosen archetype, composition, state coverage, and proof plan fit the product. Do not copy their visual treatment.

## Creative Studio: Color-Grading Workbench

Context card: `studio/editor + create + current shot/timeline + expert repeated use + multi-panel workspace + selected/unsaved/export proof`.

- Wrong reflex: dark dashboard with GPU metrics, render cards, and presets above a reduced preview.
- Direction: inspector workbench. The current frame and timeline dominate; scopes support the image; controls group by color intent and selected layer.
- Signature move: parameter-to-image tether that reveals which region or range a control affects.
- Preserve: keyboard grading speed, undo history, clip selection, comparison mode, and export settings.
- Prove: blank project, loaded timeline, selected layer, before/after comparison, unsaved and conflicting edits, collapsed panels, export failure, long clip names, and real-size icon alignment.

The studio succeeds when the artifact grows clearer while the chrome becomes more exact, not merely darker.

## Dashboard: Hospital Capacity Decision

Context card: `dashboard + decide + staffing allocation + time-sensitive uncertain data + document/evidence pane + missing/stale/forecast proof`.

- Wrong reflex: twelve bed-count KPI cards followed by unrelated chart variety.
- Direction: question-led overview plus decision table and evidence pane.
- Signature move: every ward row shows capacity, forecast interval, feed freshness, staffing gap, and the threshold that changes the decision.
- Preserve: exact values, source timestamps, filter scope, accessible tabular alternative, and export/report path.
- Prove: zero capacity, missing feed, stale feed, uncertainty, surge threshold, long ward names, filtered cohort, and narrow layout.

The dashboard succeeds when the staffing decision becomes faster and more defensible, not when chart count increases.

## Command Center: Regional Fleet Incidents

Context card: `command center + monitor/respond + active incident + continuous high-consequence use + multi-panel shell + alert-flood/handoff/recovery proof`.

- Wrong reflex: neon map, pulsing red cards, fake terminal feed, and auto-sorted alerts with no owner.
- Direction: operator cockpit with an incident spine. Map and fleet state orient; the selected incident holds freshness, scope, evidence, owner, acknowledgement, actions, and recovery.
- Signature move: recovery rail keeps failed, deferred, and reversible actions visible beside the incident.
- Preserve: keyboard triage, timestamp precision, audit trail, acknowledgement, team handoff, and partial subsystem visibility.
- Prove: normal baseline, one incident, alert flood, stale GPS source, comms outage, assignment, recovery, resolved state, wallboard mode, and operator mode.

The console succeeds when urgency is discriminating and stable. If everything glows, nothing is operationally loud.

## Browser Game: Co-op Extraction HUD

Context card: `game HUD + act/respond + world/combat focal area + continuous controller use + viewport overlay + noisy-play/input-switch proof`.

- Wrong reflex: admin cards in every corner, translucent sidebar inventory, tiny outline icons, and scrolling status panels.
- Direction: HUD perimeter during play; dedicated full-screen modes for map, inventory, and squad management.
- Signature move: objective and extraction pressure compress into one readable temporal cue rather than another card.
- Preserve: gameplay visibility, controller focus, safe areas, subtitle space, input glyph switching, color-vision-independent state, and pause behavior.
- Prove: calm traversal, combat, critical health, objective update, full squad status, controller/keyboard switch, long localization, ultrawide and narrow aspect ratios, bright/dark scenes, reduced motion, and stream overlay collision.

The HUD succeeds when players read it at a glance while watching the game, not when it looks impressive on a flat mockup.

## Interactive Lab: Water Shader Playground

Context card: `interactive prototype + create/explore + rendered water + occasional direct manipulation + full-screen output/workbench + nonblank/fallback/export proof`.

- Wrong reflex: feature landing page before the canvas or a cyber dashboard with logs louder than the effect.
- Direction: playable surface first plus compact instrument panel.
- Signature move: controls highlight the visible water region or property they affect.
- Preserve: presets, reset, reproducible parameters, performance readout, and export.
- Prove: nonblank output, meaningful slider change, extreme values, reset, WebGL fallback, reduced motion, background pause, export failure, mobile controls, and representative frame pressure.

## Landing: Architectural Lighting Studio

Context card: `landing + choose/trust + physical fixture and room effect + one-time considered purchase + scroll document + first-viewport/proof/mobile evidence`.

- Wrong reflex: beige luxury hero, oversized serif, three feature cards, and invented hospitality logos.
- Direction: artifact stage plus proof wall. Pair fixture geometry and material with the room effect it produces.
- Signature move: a light-falloff comparison that moves from object to occupied space.
- Preserve: project inquiry, service area, lead time, maintenance facts, real case material, SEO, analytics, and form behavior.
- Prove: small-laptop hero, mobile crop, missing image fallback, room proof, long project names, inquiry errors, and honest asset provenance.

## Cross-Context Test

Before approval, ask:

- Would moving this composition unchanged to another example still look plausible?
- Did the proof matrix exercise the costly moments of this product or only generic breakpoints?
- Did a secondary region quietly take over the primary archetype?
- Is the main artifact obvious with gradients, shadows, and imagery temporarily removed?

Any `yes` to the first or `no` to the others means the direction is still generic.
